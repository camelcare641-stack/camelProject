## 1. Төслийн ялгах тэмдэг

- **Төсөл:** "ТЭМЭЭ" хүүхэд хөгжлийн төсөл
- **Байгууллага:** "Дадал Тэнцвэр" Төрийн Бус Байгууллага (ТББ)
- **Уриа үг:** "Тэнцвэртэй нийгэм — Гэрэлтэй ирээдүй"
- **Гол мессеж:** "Нэг тэмээ — Нэг хүүхдийн ирээдүй"
- **Эквалент:** **25,000₮ = нэг хүүхдийн боломж**

## 2. Бид юу бүтээж байна вэ

Хандивын **нэг бүтээгдэхүүнтэй** вэб сайт. Зорилго: зочин (зочин = боломжит хандивлагч) төслийг ойлгох → итгэх → 25,000₮ төлж бэлгэдлийн "Тэмээ" авах.

- Энгийн, унших боломжтой, **гар утсанд тааруулсан** (mobile-first).
- Бүх UI текст **Монгол Кирилл**. Код, identifier, commit message **англи**.
- Бүх Монгол хуулбар (UI string) нэг газар: `src/lib/content.ts`. Component-д шууд Кирилл бичихгүй.

## 3. Төслийн үндэслэл (.docx §1)

Хотын захын хороолол болон зорилтот бүлгийн хүүхдүүдийн тулгарч буй асуудлууд:

- сэтгэлзүйн дарамт
- гэр бүлийн хүчирхийлэл
- боловсролын тэгш бус байдал
- өөртөө итгэх итгэл сул
- хөгжлийн боломж хомс
- цахим болон нийгмийн сөрөг нөлөөлөл

Эдгээр хүүхдүүдэд **зөвхөн материаллаг тусламж бус**, сэтгэлзүйн дэмжлэг, хөгжлийн орчин, хайр халамж, сонсох хүн хэрэгтэй.

## 4. Төслийн санаа — "Тэмээ" загвар (.docx §2)

Иргэн, байгууллага, сайн дурын дэмжигчид **25,000 төгрөгөөр бэлгэдлийн "Тэмээ"** (гар урлалын арьсан цүнх / түлхүүрний оосор) худалдан авна.

Энэхүү 25,000₮ нь нэг хүүхдийн дараах дэмжлэгт хүрнэ:

1. сургалт
2. сэтгэлзүйн зөвлөгөө
3. хөгжлийн үйл ажиллагаа
4. хамгааллын үйлчилгээ
5. урлаг спортын оролцоо

### "Тэмээ" бэлгэдлийн утга

тэсвэр · амьдралын хүч · холын замыг давах чадвар · халамж · тогтвортой байдал.

→ Төсөл нь хүүхдийн ирээдүйг **"хамтдаа тээх"** утгатай.

### Зүүсэн хүний үнэ цэн

"Тэмээ"-г зүүсэн хүн нь анхаарал тавьдаг, ухаалаг иргэний илэрхийлэл — ижил зорилготой хүмүүсийн хүрээлэлд нэгдэнэ.

## 5. Төслийн зорилго ба зорилт (.docx §3, §4)

**Зорилго:** Хотын захын болон зорилтот бүлгийн хүүхдүүдэд боловсрол, хөгжлийн болон сэтгэлзүйн **цогц дэмжлэг** үзүүлж, тэдний ирээдүйд **эерэг өөрчлөлт** бий болгох.

**5 зорилт:**

| # | Зорилт |
| - | ------ |
| 4.1 | Хүүхдүүдэд амьдрах ухаан, хувь хүний хөгжлийн сургалт зохион байгуулах |
| 4.2 | Сэтгэлзүйн зөвлөгөө, сэтгэл заслын дэмжлэг үзүүлэх |
| 4.3 | Эрсдэлт нөхцөлд байгаа хүүхдүүдийг нийгмийн хамгааллын үйлчилгээтэй холбох |
| 4.4 | Хүүхдийн өөртөө итгэх итгэл болон нийгмийн оролцоог нэмэгдүүлэх |
| 4.5 | Сайн дурын болон олон нийтийн оролцоог нэмэгдүүлэх |

## 12. Нүүр хуудасны 5 хэсэг (донорын шийдвэрийн дараалал)

> Энэ дараалал нь хатуу. Хэсэг бүр **нэг л асуултанд** хариулна.

1. **Hero banner** — *"Энэ юу вэ?"*
   Уриа, "Нэг тэмээ — Нэг хүүхдийн ирээдүй", тэмээний фото, **"Хандив өгөх"** товч, доор нь **live raised total (₮)** + **сүүлд хандивласан хүмүүсийн нэрс гүйдэг marquee** (Supabase-аас, шинэ → хуучин).
2. **Төслийн зорилго** — *"Яагаад надад хамаатай вэ?"*
   2–3 өгүүлбэр + 3 icon (Боловсрол · Сэтгэлзүй · Халамж) → "Дэлгэрэнгүй →" `/about`.
3. **"Тэмээ" гэж юу вэ?** — *"Юу хийх вэ, юунд хүрэх вэ?"* (конверсийн зүрх)
   Гар урлалын бүтээгдэхүүн · **25,000₮ = Нэг хүүхдийн боломж** · зүүсэн хүний нийгмийн илэрхийлэл. CTA "Хандив өгөх".
4. **Хөтөлбөрийн мэдээ** — *"Танд итгэж болох уу?"*
   Supabase `news` table-аас horizontal scroll card-ууд (зураг · гарчиг · огноо) → "Үйл ажиллагаа →" `/activities`.
5. **Хандив өгөх** — *"Яаж өгөх вэ?"*
   Банкны мэдээлэл + copy-to-clipboard · QR код · "хэдэн тэмээ" calculator (qty × 25,000₮ → live total) · "**гүйлгээний утганд нэр + утас бичих**" заавар.

## 13. Технологийн стек

| Давхарга | Сонголт |
| -------- | ------- |
| Frontend / backend | **Next.js 16** (App Router, TypeScript, `src/` layout). ⚠ Энэ нь шинэ хувилбар — API өөрчлөгдсөн. `node_modules/next/dist/docs/`-ыг уншиж байж код бичнэ. |
| DB / auth / realtime | **Supabase** (Postgres, Tokyo `ap-northeast-1` region) |
| UI | **Tailwind CSS v4** + **shadcn/ui** primitives |
| Validation | **Zod** |
| Төлбөр | **QPay v2** (Монголын merchant API), Next.js API route-аар |
| Deploy | **Vercel** |
| Фонт | Alegreya (heading), Manrope (body), Caveat (accent) — бүгд Кирилл дэмждэг |

### Supabase client-ийн 3 хувилбар

- `lib/supabase/client.ts` — browser, anon key
- `lib/supabase/server.ts` — server components / route handlers, anon + cookies
- `lib/supabase/admin.ts` — **service-role**, `import "server-only"`, зөвхөн **QPay webhook** ашиглана

### QPay-ийн чухал зүйлс

- Webhook **idempotent**: `donations.qpay_payment_id` нь `UNIQUE`; callback нь `.eq('status', 'pending')` ашиглан давхар credit-ээс хамгаална.
- Local dev-д `ngrok http 3000` (эсвэл `cloudflared`) ашиглан public callback URL гаргана.

## 14. Дизайны систем — "Gobi Warm" палитр

Дулаахан, шороон өнгөтэй, дотно. Цагаан зайгаар тэнцвэр барина.

| Үүрэг | Нэр | Hex |
| ----- | --- | --- |
| Дэвсгэр (~70%) | White | `#FFFFFF` |
| Лого, гарчиг, link | Clay | `#9C4F22` |
| **Зөвхөн** "Хандив өгөх" товч | Desert teal | `#0F6E56` |
| Чимэглэлийн өргөлт | Gold | `#C8923A` |
| Том impact тоо | Deep gold | `#A2741F` |
| Body текст | Charcoal | `#2E2A26` |
| Section band — neutral | Warm surface | `#F6F4EE` |
| Section band — brand | Clay tint | `#F3E6DC` |
| Section band — confirmation | Teal tint | `#E1F0EB` |
| Section band — highlight | Gold tint | `#F7EBD3` |
| Hover (clay) | Clay dark | `#7E3F1A` |
| Hover (teal) | Teal dark | `#0A5141` |
| Зураас, хүрээ | Border | `#E6E0D4` |

### Өнгөний дүрэм (хатуу баримтлана)

- **Цагаан** layout-ыг тээнэ — экран бүрийн ~70%. Whitespace л сайтыг professional харагдуулдаг.
- **Clay** = identity. Лого, heading, link дээр хэмжээтэй ашиглана.
- **Teal**-ийн **цорын ганц үүрэг**: "Хандив өгөх" товч ба primary action. Өөр газар teal харагдвал хандивын товч "тусгай" гэдгээ алдана.
- **Gold** зөвхөн чимэглэл. Текстэнд хэзээ ч бүү ашигла (цагаан дэвсгэр дээр WCAG-ийг давдаггүй). **Deep gold** зөвхөн **том impact тоо** (raised total гэх мэт).
- Өнгөтэй дэвсгэр дээр **зөвхөн цагаан / charcoal** текст. Өнгөтэй текстийг өнгөтэй дэвсгэр дээр **хэзээ ч бүү тавь**.

## 15. Өгөгдлийн загвар (Supabase)

`supabase/migrations/` дотор:

1. `0001_schema.sql` — table, index, view
2. `0002_rls.sql` — Row Level Security policy
3. `0003_seed.sql` — анхны campaign + жишээ агуулга

### Гол table-ууд (BUILD_PROMPT-ийн дагуу)

```sql
donors    (id, name, amount default 25000, created_at)
news      (id, title, summary, image_url, published_at)
messages  (id, name, contact, topic, body, created_at)
donations (... QPay-той холбоотой бүрэн схем 0001_schema.sql дотор)
```

### RLS дүрэм

- `donors`, `news`: public **read**.
- `messages`: public **insert only**, read хаалттай.
- `donations`: admin-only. Public ticker, transparency хуудас зөвхөн **`public_donations` view**-аас уншина (зөвхөн `donor_name, amount, message, paid_at` зөвшөөрөгдсөн, төлсөн + non-anonymous донор).

### Realtime

- Donor ticker нь `donations` broadcast channel, `paid` event-ийг сонсоно.
- Postgres trigger дээр `realtime.send(...)` нэмэх ажил **дутуу** — `src/hooks/use-donations-realtime.ts` дотор TODO бий.
